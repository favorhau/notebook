# 结构区分

在互联网上常常可以查询的到两种数据的差别，根本性的差异就是

Mongodb是非关系型数据库（Nosql） 11 而mysql是关系型数据库（sql）

在传统的互联网站点上，许多数据的存储更多的是静态的，而网站的访问量一般都不会太大，使用单个数据库可以去解决应付这样的一些问题。这时候Nosql就产生了，Nosql不需要很严格的数据类型，往往

非关系型数据库里面的结构跟关系型数据库的差别往往体现于：

SQL术语/概念 MongoDB术语/概念 解释/说明
database database 数据库
table collection 数据库表/集合
row document 数据记录行/文档
column field 数据字段/域
index index 索引
table joins 表连接,MongoDB不支持
primary key primary key 主键,MongoDB自动将_id字段设置为主键

SQL 使用tables进行数据储存，相应的

Mon­goDB, on the other hand, is a non-relational database that stores data as ‘documents’ in a binary representation called BSON (Binary JSON)

NoSQL采用document，也标识为BSON

# 实际场景当中的使用

NoSQL databases were created in response to the limitations of traditional relational database technology. When compared to relational databases (opens new window), NoSQL databases are often more scalable and provide superior performance. In addition, the flexibility and ease of use of their data models can speed development in comparison to the relational model, especially in the cloud computing environment.

面对大规模的计算，简单的单一程序配置单一的数据库显然无法完成需求，

在互联网初期的时候，数据量相对较小，在许多情况下进行全表查询的用时并不多，完全可以依赖单数据库储存
