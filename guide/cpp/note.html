# Numpy.stack

# 官方解释

官方对于numpy.stack的解释为

Join a sequence of arrays along a new axis.

The axis parameter specifies the index of the new axis in the dimensions of the result. For example, if axis=0 it will be the first dimension and if axis=-1 it will be the last dimension.

New in version 1.10.0.

Parameters: arrays : sequence of array_likeEach array must have the same shape.axis : int, optionalThe axis in the result array along which the input arrays are stacked.
Returns: stacked : ndarray The stacked array has one more dimension than the input arrays.

# 个人理解

按照官方文档，顾名思义就是往某一个轴完成堆叠，如果在机器学习中，如果超过了三维，那空间结构就难以想象。所以我在想有没有一个通用的解释方式，能够解释该函数的使用呢？

第一个引入思维的应该是在吴恩达老师机器学习python的学习笔记当中的注释：

# The first axis (axis=0) refers to rows (training examples) 
# and second axis (axis=1) refers to columns (features).

假设有一个数据集

	Population	Profit
0	6.1101	17.5920
1	5.5277	9.1302
2	8.5186	13.6620
3	7.0032	11.8540
4	5.8598	6.8233

进行数据处理后

#X=[6,1101,5.5277,8,5186,7.0032,5.8598]
#Y=[17,5920,9.1302,13.6620,11,8540,6.8233]

如果设置axis=1，则得到的数组为

#array([[6.0000e+00 1.7000e+01][1.1010e+03 5.9200e+03][5.5277e+00 9.1302e+00][8.0000e+00 1.3662e+01][5.1860e+03 1.1000e+01][7.0032e+00 8.5400e+03][5.8598e+00 6.8233e+00]])

这样所得到的二维数组就包含了每一个example的features

因此比起通过矩阵和坐标认识np.stack函数，通过数组认识才是最佳。

先来看一个demo

假设两个二维数组

a =np.array([[1,2,3],[4,5,6]])#shape:(2,2)
b =np.array([[2,4,6],[8,10,12]])#shape:(2,2)

对其axis=0堆叠得到的数组为


[[[ 1  2  3]
  [ 4  5  6]]

 [[ 2  4  6]
  [ 8 10 12]]]#shape:(2,2,3)

首先对于任意维度的Array可以看作

[Array,Array,Array...]

然而大的Array里面又有小的Array

因此，对于axis这个参数，就能有了很好的理解。

首先对于axis=0我们可以视作为对最外层的Array进行堆叠

对于axis=1可以视作为对最次于外层的Array进行堆叠

shape参数亦是从左到右

以此类推

来一个二维数组的例子

a=Array([[1,2],[1,3],[5,1]])#shape:(3,2)或(3,2,1)
b=Array([[5,2],[6,3],[7,1]])#shape:(3,2)或(3,2,1)

可以知道最外层的数组为[[1,2],[1,3],[5,1]],次于这一层的数组为[1,2],[1,3],[5,1]，再次于的是1,2,1,3,5,1,因此，

output = np.stack([a,b], axis=0)
print(output)
#output:
[[[1,2],[1,3],[5,1]],[[5,2],[6,3],[7,1]]]

axis=0即对最外层的array进行堆叠，形成了一个三维数组

相关的思想可以推广至多维数组，从数组的角度理解比从矩阵的角度理解更好。
